What is Vim?

  • Free Software text editor
  • Very powerful, used by programmers
  • Ubiquitous - on most servers
  • Awesome when you know it, but can be a learning curve

  • Wordprocessors: Better for layout.
  • IDEs: XCode (iTunes for code)
  • Lightweight editors: Notepad, nano, gedit
  • Programmer’s Editors: emacs, vim, sublime text

Why Vim?

Horms and I discussed this, and we came up with two categories of people. People who use Vim, and people who don’t realize that they could be using Vim.

Potted history

Stands for VI iMproved. vi invented 1976. Screen mode for a "line based editor" called ex. Vim created 1991. GPL licenced. Asks users to donate to charity in Uganda if they like it.

GNU/Linux: apt-get install vim

OS-X: https://code.google.com/p/macvim/

Whingedoze: http://vim.org/download.php

Using Vim is like talking to your editor in ‘verb modifier object’ sentences, turned into acronyms

Yan Pritzker in Learn to speak vim – verbs, nouns, and modifiers!

Tenses (modes)

  • Normal
  • Visual
  • Command
  • Insert/Replace

Changing tense

iI (insert before, after), oO (new line before, after), aA (append, end of line)
Esc: Normal mode from insert mode
:command mode. We focus on normal mode.

Movements (chars)

h,j,k,l (or arrows).
You won’t be able to use the arrows in vi. You have to move your hand away from the keyboard.

Movements (words)

w,b move a word forward/back at a time. Depending on what you mean by a word. You can also use W,B if you think of a word as anything except for a space

Movements (lines)

$ and g_ end and last char of line. 0 and ^ start and first char of line

Movements (blocks)

() start/end of sentence
{} start/end of paragraph
% matching bracket

Movements (screens)

H,M,L top, middle, bottom of screen
PgUp, PgDown (ctrl-U and ctrl-D for half pages)

Movements (files)

gg: first line; G last line; G122 go to line 122

Unspeaking and repeating

u: undo, ctrl-r: redo

First verbs

  • y: yank (like copy)
  • d: delete (like cut)
  • c: change (like delete/replace)
  • p: paste
  • v: visually select

Capitalize verbs to speak about lines (P pastes before cursor).

Movements can be nouns

  • w: word
  • j: down
  • $: til end of line

Let’s make a sentence

  • yw: yank word
  • dj: delete from here down
  • d$: delete from here til end of line

Don’t forget, you can u to undo!

Some more nouns

  • as: a sentence
  • ap: a paragraph
  • at: a tag
  • is/ip/ib/iw: inner paragraph, sentence, word, etc
  • Inside or Around text unit. Sentence, para, tag always take an i or a, and need an action.

More sentences

  • dap: delete a paragraph
  • ciw: change inside word
  • yi(: yank between parentheses

You can combine phrases dawciw

Single character verbs

  • ~: change case of char
  • x: delete char
  • r [char2]: replace char with char2

RepetitionRepetition

What if I want to delete two words then? Just prepend a count to any phrase

  • 2dw: Delete 2 words
  • 4Y: Yank 4 lines
  • 5dap: Delete 5 paragraphs

The best key in Vim?

. : Repeat the last phrase. eg. daw to delete a word, . to delete another word

Searchy movements

  • f or F[char]: search "forward" or backward in line (inc char)
  • t or T[char]: search "til" or "til" backward in line (exc char)
  • /: search for a word or regex (regexes are complicated)

Some more sentences

  • dfJ: delete line upto and including J
  • dt<: delete line til next <
  • 2dtQ: delete line til the second Q
  • d/XYZ: delete everything til the next XYZ

You have to press enter after a / search

Searching like a Ninja

  • You can use regexes(wildcards) in a / search.
  • You can use * to search forwards for word under cursor (# back). n/N to move between occurences
  • Command mode :set hls or :set nohls to highlight your search time

Search and replace

Search and replace happens in command mode. Line orientated, but you can put % before it for whole file

:s/this/that/[g]

Command mode: Files

  • :wq write and quit
  • :sav newfile "save as"
  • :q! quit without saving

Command mode: Tabs/Splits

  • :tabnew [filename]
  • :sp [filename]
  • :vsp [vertical split]

Command mode: The shell

  • :! cmd - run smd in a shell
  • :!r cmd - run in shell and read back at cursor
  • :[10,20]! cmd - pipe lines 10 to 20 to cmd
  • Bonus trick: ctrl-z to hop into your teminal, fg to get back
